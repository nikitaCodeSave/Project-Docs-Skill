# Discovery Log: Audio Transcription Service

> Этот файл — пример лога discovery-разговора. Демонстрирует процесс выявления требований до написания PRD.

## Контекст разговора

**Дата:** 2024-01-15
**Участники:** Заказчик (разработчик/аналитик), AI

---

## Ключевые ответы

### Проблема

**Q:** Какую проблему решаем?

> "Облачные сервисы транскрипции требуют отправки данных на внешние сервера — неприемлемо для конфиденциальных записей совещаний и интервью."

**Q:** Что происходит, если проблему не решить?

> "Приходится транскрибировать вручную или рисковать утечкой данных."

**Q:** Как проблема решается сейчас?

> "Иногда использую OpenAI Whisper через API, но это тоже облако. Локальный Whisper пробовал — работает, но неудобно."

### Аудитория

**Q:** Кто будет использовать?

> "Разработчики и аналитики. Работают из терминала, не нужен GUI."

**Q:** Их технический уровень?

> "Комфортны с CLI. Не хотят разбираться с настройкой моделей — просто `stt file.mp3` и готово."

### Scope и приоритеты

**Q:** Каков минимальный scope для MVP?

> "Для начала: один аудиофайл на входе, текст + разделение по спикерам на выходе. Не нужен batch, не нужен realtime."

**Q:** Какие функции критичны vs nice-to-have?

> "Критично: транскрипция + diarization. Nice-to-have: поддержка разных форматов вывода, batch-обработка."

**Q:** Что точно НЕ нужно делать?

> "Web-интерфейс, cloud deployment, обработка видео."

### Технический контекст

**Q:** Есть ли предпочтения по технологиям?

> "Python предпочтительно. GPU есть — RTX 3080, 8GB VRAM."

**Q:** Какие интеграции планируются?

> "Пока никаких. Возможно, позже — скриптование в пайплайнах."

**Q:** Ограничения?

> "Только локальная обработка. Никаких внешних API."

### Использование результата

**Q:** Как будет использоваться?

> "CLI-first. Запуск из терминала, результат в файл или stdout."

**Q:** Какой формат вывода нужен?

> "JSON с таймкодами + plain text. SRT было бы плюсом, но не критично для MVP."

### Ключевые решения для PRD

- **Формат:** CLI-first
- **Ограничение:** только локальная обработка
- **MVP-фокус:** один файл → текст + diarization
- **Технология:** Python + GPU

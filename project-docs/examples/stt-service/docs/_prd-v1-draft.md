# PRD v1 (Draft): Audio Transcription Service

> **Это черновик** — демонстрирует итеративный процесс. После feedback от заказчика scope был сужен. Финальная версия: [PRD.md](PRD.md)

## Обзор

### Проблема

Облачные сервисы транскрипции требуют отправки данных на внешние сервера, что неприемлемо для конфиденциальных записей.

### Решение

CLI-инструмент для локальной транскрипции аудио с разделением по спикерам.

## Цели проекта

| ID | Цель |
|----|------|
| G1 | Качественная транскрипция русской речи |
| G2 | Разделение по спикерам (diarization) |
| G3 | Полная приватность — только локальная обработка |

## Scope (v1 — до feedback)

### В рамках проекта

- [x] Транскрипция аудиофайлов
- [x] Diarization
- [x] CLI интерфейс
- [ ] **Batch-обработка нескольких файлов** ← *убрано после feedback*
- [ ] **Поддержка 5 форматов вывода** ← *сужено до 2*
- [ ] **Конфигурация через YAML** ← *убрано, достаточно CLI-флагов*

### Вне рамок

- Web-интерфейс
- Cloud deployment
- Realtime транскрипция

## Feedback от заказчика

**Дата review:** 2024-01-16

### Комментарии

> "Batch не нужен для MVP — сначала сделаем хорошо для одного файла."

> "5 форматов — overkill. JSON + plain text достаточно. SRT может подождать."

> "YAML-конфиг лишний. CLI-флаги проще."

### Решения

| Пункт | Было | Стало | Причина |
|-------|------|-------|---------|
| Batch | In Scope | v2 | Фокус на core-функциональности |
| Форматы вывода | 5 | 2 (JSON, TXT) | Достаточно для MVP |
| YAML-конфиг | In Scope | Out of Scope | CLI-флаги проще |

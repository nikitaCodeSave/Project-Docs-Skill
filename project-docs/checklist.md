# Чеклист: Готовность документации к разработке

## Минимальный набор (MVP)

### CLAUDE.md
- [ ] Краткое описание проекта (1-2 предложения)
- [ ] Контекст предметной области
- [ ] Таблица ссылок на docs/ с описанием "когда читать"
- [ ] Ключевые решения (если есть ADR)
- [ ] Технологический стек (зафиксированные технологии)
- [ ] Критичные ограничения
- [ ] Секция "DO NOT" с границами для AI

### docs/PRD.md
- [ ] Проблема чётко сформулирована
- [ ] Решение описано (высокоуровнево)
- [ ] Целевая аудитория определена
- [ ] Цели конкретны и измеримы
- [ ] Каждая цель связана с SPEC (трассируемость)
- [ ] Метрики успеха с числовыми значениями
- [ ] Scope: что делаем и что НЕ делаем
- [ ] Ограничения и допущения

### docs/SPEC.md
- [ ] Глоссарий терминов домена
- [ ] Допущения явно указаны
- [ ] Все FR описаны
- [ ] Каждый FR имеет acceptance criteria (GIVEN/WHEN/THEN)
- [ ] Каждый FR связан с целями PRD
- [ ] NFR определены
- [ ] Ограничения системы перечислены
- [ ] "Вне рамок версии" указано

## Расширенный набор

### docs/api.md (если CLI/REST)
- [ ] Все команды/endpoints описаны
- [ ] Аргументы и опции указаны
- [ ] Copy-paste ready примеры
- [ ] Коды возврата/ошибок

### docs/data-formats.md (если сложные данные)
- [ ] Входные форматы с ограничениями
- [ ] Выходные форматы с JSON Schema
- [ ] Примеры для каждого формата

### docs/decisions/ADR-*
- [ ] Ключевые решения зафиксированы
- [ ] Альтернативы описаны
- [ ] Обоснование выбора
- [ ] Последствия указаны

---

## Красные флаги ❌

**В PRD:**
- Неизмеримые цели
- Размытый scope
- Технические детали реализации

**В SPEC:**
- Рабочий код
- Имена классов/функций
- "КАК" вместо "ЧТО"
- AC без GIVEN/WHEN/THEN

**В CLAUDE.md:**
- Более 2 экранов
- Нет ссылок на docs/
- Нет границ для AI

---

## Быстрая проверка

1. Claude Code поймёт, что делает проект? (CLAUDE.md + PRD)
2. Claude Code поймёт, что реализовать? (SPEC + AC)
3. Claude Code поймёт, чего НЕ делать? (границы, out of scope)
4. Claude Code поймёт, почему выбраны технологии? (ADR)
5. Можно протестировать каждое требование? (AC)
6. Термины домена однозначны? (глоссарий)

Если хотя бы один ответ "нет" — документация неполна.

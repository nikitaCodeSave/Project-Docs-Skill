## Красные флаги

**В PRD:**
- Неизмеримые цели ("улучшить", "быстрее")
- Размытый scope
- Технические детали реализации

**В SPEC:**
- Рабочий код
- Имена классов/функций
- "КАК" вместо "ЧТО"
- AC без GIVEN/WHEN/THEN

**В CLAUDE.md:**
- Более 2 экранов
- Нет ссылок на docs/
- Нет границ для AI (DO NOT)

## Финальная проверка

### CLAUDE.md (создаётся последним)
- [ ] Краткое описание проекта (1-2 предложения)
- [ ] Таблица ссылок на docs/ с описанием "когда читать"
- [ ] Технологический стек (на основе ADR)
- [ ] Ключевые решения со ссылками на ADR
- [ ] Контекст предметной области
- [ ] Секция DO NOT с границами для AI
- [ ] Критичные ограничения

### Контрольные вопросы

1. Claude Code поймёт, что делает проект? (CLAUDE.md + PRD)
2. Claude Code поймёт, что реализовать? (SPEC + AC)
3. Claude Code поймёт, чего НЕ делать? (границы, out of scope)
4. Claude Code поймёт, почему выбраны технологии? (ADR)
5. Можно протестировать каждое требование? (AC)
6. Термины домена однозначны? (глоссарий)

Если хотя бы один ответ "нет" — документация неполна.
